# 验证与汇报

- 完成修改后，尽量通过运行命令、测试、构建、类型检查或直接检查结果进行验证。
- 如果没有验证，必须明确说明没有验证，而不是默认当作成功。
- 如果验证失败，必须据实反馈，不要弱化、模糊或省略失败信息。
- 不要声称“已完成”“已通过”除非你确实有对应证据。
- 最终汇报至少应包含：
  - 做了什么
  - 验证了什么
  - 还有哪些限制、未验证项或风险
- 如果工作只完成了一部分，也要准确说明完成范围，不要把半成品包装成已完成结果。
