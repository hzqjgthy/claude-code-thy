# 功能测试手册

这份文档只按一种方式写：

- 进入 `claude-code-thy` 项目目录
- 直接输入测试命令
- 看会出现什么结果

不再用临时目录，也不要求你在终端里设置任何环境变量。

## 1. 进入项目目录

先进入项目根目录：

```bash
cd /Users/thy/Desktop/thy_skills/Claude_Code/claude-code-thy
```

如果你还没安装当前源码版本，先执行：

```bash
pip install --no-cache-dir --force-reinstall .
```

## 2. 先确认主程序能跑

测试命令：

```bash
claude-code-thy --help
```

会看到：

- 主命令帮助
- `mcp` 子命令入口

再测试：

```bash
claude-code-thy --print "你好"
```

会看到：

- 一条模型回答

如果这里提示 provider 未配置，说明你项目根目录的 `.env` 还没有配好。先按 README 把 `.env` 配好，再继续往下测。

## 3. 先准备几个测试文件

下面这些命令都在 `claude-code-thy` 根目录执行。

### 3.1 准备普通测试文件

```bash
mkdir -p manual-test
mkdir -p manual-secret

cat > manual-secret/token.txt <<'EOF'
super-secret-token
EOF
```

作用：

- 后面用来测试 `/write`、`/edit`
- 后面用来测试路径权限确认

### 3.2 准备 skill 测试文件

如果你已经有自己的 `.claude-code-thy/skills`，可以先备份。没有的话直接执行下面这些命令。
当前项目默认 skill 根目录只认 `.claude-code-thy/skills`，不再使用旧的 `.claude/skills`。

```bash
mkdir -p .claude-code-thy/skills/review
mkdir -p .claude-code-thy/skills/inspect
mkdir -p .claude-code-thy/skills/python-review
mkdir -p .claude-code-thy/skills/hidden-user
mkdir -p .claude-code-thy/skills/no-model
mkdir -p skill-packs/triggered-review
```

创建 `review`：

```bash
cat > .claude-code-thy/skills/review/SKILL.md <<'EOF'
---
description: 评审一个主题
arguments:
  - topic
allowed-tools:
  - read
  - grep
---
请认真评审 ${topic}，并列出主要风险点。
EOF
```

创建 `inspect`：

```bash
cat > .claude-code-thy/skills/inspect/SKILL.md <<'EOF'
---
description: 使用 fork agent 检查目标
arguments:
  - target
context: fork
allowed-tools:
  - read
  - glob
  - grep
effort: medium
---
请检查 ${target}，并返回一段简洁总结。
EOF
```

创建 `python-review`：

```bash
cat > .claude-code-thy/skills/python-review/SKILL.md <<'EOF'
---
description: 评审当前会话触碰到的 Python 文件
arguments:
  - topic
paths:
  - src/*.py
---
请围绕 ${topic} 评审当前触碰到的 Python 文件。
EOF
```

创建 `hidden-user`：

```bash
cat > .claude-code-thy/skills/hidden-user/SKILL.md <<'EOF'
---
description: 不显示在 slash 命令列表中
user-invocable: false
---
这个 skill 不应该出现在 /skills 列表里。
EOF
```

创建 `no-model`：

```bash
cat > .claude-code-thy/skills/no-model/SKILL.md <<'EOF'
---
description: 只允许用户直接调用的 skill
disable-model-invocation: true
---
这是一个只允许用户直接调用的 skill，参数是：${args}。
EOF
```

创建 `triggered-review`：

```bash
cat > skill-packs/triggered-review/SKILL.md <<'EOF'
---
description: 位于默认根目录之外的触发式评审 skill
arguments:
  - topic
---
请对 ${topic} 执行触发式评审。
EOF
```

### 3.3 准备权限测试配置

如果你已经有 `.claude-code-thy/settings.local.json`，可以先备份。没有的话直接创建。

```bash
cat > .claude-code-thy/settings.local.json <<'EOF'
{
  "permissions": [
    {
      "effect": "ask",
      "tool": "bash",
      "target": "command",
      "pattern": "echo permission-test",
      "description": "测试命令权限确认"
    },
    {
      "effect": "ask",
      "tool": "read",
      "target": "path",
      "pattern": "/Users/thy/Desktop/thy_skills/Claude_Code/claude-code-thy/manual-secret/*",
      "description": "测试路径权限确认"
    }
  ],
  "skills": {
    "enabled": true,
    "triggers": [
      {
        "pattern": "src/*.py",
        "skill_dir": "skill-packs/triggered-review"
      }
    ]
  }
}
EOF
```

## 5. MCP 管理命令

这部分直接在项目根目录测试。

### 测试查看当前 MCP 配置

命令：

```bash
claude-code-thy mcp show-config
```

会看到：

- 当前 `.mcp.json` 内容

### 测试新增一个 self_mcp

命令：

```bash
claude-code-thy mcp add self_mcp claude-code-thy -t stdio --arg mcp --arg serve --description "loopback stdio server"
```

会看到：

- `self_mcp` 写入成功

### 测试列出 MCP

命令：

```bash
claude-code-thy mcp list
```

会看到：

- 当前 MCP server 列表
- 重点看里面有没有 `self_mcp`

说明：

- 如果你本来配置过别的 MCP，比如 `xiaohongshu-mcp`，它出现在列表里也正常
- 如果别的服务没启动，它可能显示失败，但这不影响你看 `self_mcp`

### 测试查看单个 MCP

命令：

```bash
claude-code-thy mcp get self_mcp
```

会看到：

- `self_mcp` 的详细信息
- 工具列表

### 测试移除 MCP

如果你想把刚才加的 `self_mcp` 删除，执行：

```bash
claude-code-thy mcp remove self_mcp
```

会看到：

- `self_mcp` 被移除

如果后面你还要做 MCP 动态工具测试，就先不要删，留到最后再删。

## 11. MCP 动态工具

/mcp__xiaohongshu_mcp__check_login_status

先确保你前面已经加过 `self_mcp`，并且它能在 `/mcp` 里看到。

### 测试动态 MCP `read`

需要先在 .mcp.json 中配置自己：
"self_mcp": {
      "type": "stdio",
      "description": "loopback stdio server",
      "command": "claude-code-thy",
      "args": [
        "mcp",
        "serve"
      ]
    }
（这样，本项目既做mcp的serve，又做client）

输入：

```text
/mcp__self_mcp__read {"file_path":"README.md"}
```

会看到：

- 返回 `README.md` 内容

### 测试动态 MCP `grep`

输入：

```text
/mcp__self_mcp__grep {"pattern":"def main","path":"src/claude_code_thy","output_mode":"content"}
```

会看到：

- 搜索结果

### 测试动态 MCP 参数错误

输入：

```text
/mcp__self_mcp__read README.md
```

会看到：

- 提示 MCP 动态工具参数必须是 JSON object

### 测试动态 MCP 工具不存在

输入：

```text
/mcp__self_mcp__does_not_exist
```

会看到：

- `MCP 动态命令未命中`
- MCP 当前状态诊断信息

### 测试动态 MCP 资源列表

输入：

```text
/mcp__self_mcp__list_mcp_resources {}
```

会看到：

- MCP resources 列表
- 如果显示 `No resources found` 也正常，说明命令已经执行到了，只是当前没有资源
