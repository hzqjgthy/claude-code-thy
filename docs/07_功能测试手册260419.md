# 功能测试手册

这份文档按当前项目的真实形态来写：

- 默认交互方式已经不是旧的终端 TUI
- 当前主要测试入口是：
  - Web API
  - Web 前端
  - 无头 CLI

所有命令都默认在这个目录执行：

```bash
cd /Users/thy/Desktop/thy_skills/Claude_Code/claude-code-thy
```

---

## 1. 先准备环境

如果你还没安装当前源码版本，先执行：

```bash
python -m pip install -e .
```

前端依赖执行一次：

```bash
cd frontend
npm install --cache .npm-cache
cd ..
```

如果 `claude-code-thy --print "你好"` 直接报 provider 未配置，说明项目根目录的 `.env` 还没配好。先按 README 配好 `.env`，再继续往下测。

## 2. 准备测试文件

### 2.1 普通文件

```bash
mkdir -p manual-test
mkdir -p manual-secret

cat > manual-test/note.txt <<'EOF'
hello from manual test
EOF

cat > manual-secret/token.txt <<'EOF'
super-secret-token
EOF
```

### 2.2 准备 skill 测试文件

当前项目默认本地 skill 根目录只认：

```text
.claude-code-thy/skills
```

执行：

```bash
mkdir -p .claude-code-thy/skills/review
mkdir -p .claude-code-thy/skills/inspect
mkdir -p .claude-code-thy/skills/python-review
mkdir -p .claude-code-thy/skills/hidden-user
mkdir -p .claude-code-thy/skills/no-model
mkdir -p skill-packs/triggered-review
```

创建 `review`：

```bash
cat > .claude-code-thy/skills/review/SKILL.md <<'EOF'
---
description: 评审一个主题
arguments:
  - topic
---
请认真评审 ${topic}，并列出主要风险点。
EOF
```

创建 `inspect`：

```bash
cat > .claude-code-thy/skills/inspect/SKILL.md <<'EOF'
---
description: 检查目标并输出简洁总结
arguments:
  - target
---
请检查 ${target}，并返回一段简洁总结。
EOF
```

创建 `python-review`：

```bash
cat > .claude-code-thy/skills/python-review/SKILL.md <<'EOF'
---
description: 评审 Python 相关主题
arguments:
  - topic
---
请围绕 ${topic} 评审 Python 相关内容。
EOF
```

创建 `hidden-user`：

```bash
cat > .claude-code-thy/skills/hidden-user/SKILL.md <<'EOF'
---
description: 不显示在 slash 命令列表中
user-invocable: false
---
这个 skill 不应该出现在 /skills 列表里。
EOF
```

创建 `no-model`：

```bash
cat > .claude-code-thy/skills/no-model/SKILL.md <<'EOF'
---
description: 只允许用户直接调用的 skill
disable-model-invocation: true
---
这是一个只允许用户直接调用的 skill，参数是：${args}。
EOF
```

创建额外搜索根目录里的 skill：

```bash
cat > skill-packs/triggered-review/SKILL.md <<'EOF'
---
description: 位于额外 skill 搜索根目录中的评审 skill
arguments:
  - topic
---
请对 ${topic} 执行额外目录中的评审。
EOF
```

### 2.3 准备权限测试配置

```bash
cat > .claude-code-thy/settings.local.json <<'EOF'
{
  "permissions": [
    {
      "effect": "ask",
      "tool": "bash",
      "target": "command",
      "pattern": "echo permission-test",
      "description": "测试命令权限确认"
    },
    {
      "effect": "ask",
      "tool": "read",
      "target": "path",
      "pattern": "/Users/thy/Desktop/thy_skills/Claude_Code/claude-code-thy/manual-secret/*",
      "description": "测试路径权限确认"
    }
  ],
  "skills": {
    "enabled": true,
    "search_roots": ["skill-packs"]
  }
}
EOF
```

## 13. 直接测 Web API

把前面 `/status` 里记下来的 `<session_id>` 换进去。

### 拉取 transcript

```bash
curl http://127.0.0.1:8002/api/sessions/<session_id>/messages
```

会看到：

- `session`
- `messages`

### 非流式聊天

```bash
curl -X POST http://127.0.0.1:8002/api/chat \
  -H 'Content-Type: application/json' \
  -d '{
    "session_id": "<session_id>",
    "prompt": "你好",
    "stream": false
  }'
```

会看到：

- `session`
- `new_messages`

### SSE 聊天

```bash
curl -N -X POST http://127.0.0.1:8002/api/chat \
  -H 'Content-Type: application/json' \
  -d '{
    "session_id": "<session_id>",
    "prompt": "请读取 README.md",
    "stream": true
  }'
```

会看到：

- `tool_event`
- `message`
- `done`

### 挂起权限查询

先在前端触发一次需要确认的命令后，再执行：

```bash
curl http://127.0.0.1:8002/api/sessions/<session_id>/pending-permission
```

会看到：

- 当前挂起的权限请求，或者 `null`

### 权限确认 API

```bash
curl -X POST http://127.0.0.1:8002/api/sessions/<session_id>/pending-permission/resolve \
  -H 'Content-Type: application/json' \
  -d '{
    "approved": true
  }'
```

会看到：

- 一次新的 `turn` 结果

### 右侧面板数据 API

```bash
curl http://127.0.0.1:8002/api/sessions/<session_id>/tools
curl http://127.0.0.1:8002/api/sessions/<session_id>/skills
curl http://127.0.0.1:8002/api/sessions/<session_id>/mcp
curl http://127.0.0.1:8002/api/sessions/<session_id>/tasks
```

会看到：

- 对应结构化 JSON
